# CHIMERA — Конструктор существ: инструкция

> Как устроена система «шасси + аугументы» и как через неё добавлять виды, органы, способности и эффекты.
> Актуальна после рефактора существ (спека: `Docs/superpowers/specs/2026-07-01-creature-abstraction-design.md`).

## Большая картина — три закона

1. **ТЕЛО строится единообразно** у всех существ: шасси + органы (аугументы) → статы + способности + восприятие. Тело — это **данные** (`SpeciesSO` + `CreatureBody`).
2. **ПСИХИКА пишется индивидуально** под вид — это **код**, но тонкий: только решения (sense → think → act) поверх «глаголов тела». Психика не переизобретает навигацию/телеграф/приёмы.
3. **СПОСОБНОСТЬ = ДОСТАВКА + ЭФФЕКТЫ.** Доставки (укус-конус, прыжок…) переиспользуются между существами с разными числами; эффекты — общий словарь, ортогональный доставкам.

Фикция = архитектура: **способность живёт на органе**, потому что химеризация = кража органа. Числа органов записаны в **мутагенной шкале** — насколько они раскрыты в конкретном теле, решает **экспрессия**.

## Словарь

| Термин | Что это | Где живёт |
|---|---|---|
| **Вид** | Ассет: имя, тинт, пул мутагена, органы. Две роли: ШАССИ (слоты + дефолт-органы) и ДОНОР (источник аугументов) | `SpeciesSO`, ассеты в `Assets/_Chimera/Data/` |
| **Орган** | Вклад в статы своего слота + флаги способностей (`enablesBite/Scent/Kick/Howl`) + цена в пуле | класс `Organ` внутри `SpeciesSO.cs` |
| **Экспрессия** | Насколько раскрыты гены зверя в теле. Игрок — авто от родства (скидка 0–80, мощь 80–100 → ×1…×2); вервольф — фикс 2; природный волк — 0.45 (зверь без сыворотки) | поле `expression` на `CreatureBody` |
| **Пул мутагена** | Ёмкость тела; каждый слот занимает пул; химеризация = размен в ёмкости | `SpeciesSO.mutagenPool` |
| **Тело** | Универсальный компонент сборки: у игрока динамика (Tab/хоткеи), у NPC — застывшая конфигурация | `CreatureBody` |
| **Доставка** | Как приём достигает цели. ИИ: `BiteAbility`, `LeapAbility` (замах через базу `WindupAbility`). Игрок: `PlayerAttack/Bite/Kick/Howl` (контракт `IAbility.TryUse`) | `Scripts/Combat/`, `Scripts/Player/` |
| **Эффект удара** | Что происходит при попадании: `Damage, LifeSteal, Knockback, RegenDebuff, Stun` | `Hit.cs` (enum + одна ветка на эффект) |
| **Статусы** | `Stagger` — два калибра: стаггер <1с (боёвка, от попаданий) и стан ≥1с (эффект). `Rage` — ярость: +урон +скорость, −защита, бесстрашие | `Scripts/Combat/` |

## Кто на чём собран (три существа)

| | Шасси | Доноры | Экспрессия | Витальность | Водитель |
|---|---|---|---|---|---|
| **Игрок** | Человек | [Волк] | авто (родство) | из органов | `PlayerInputDriver` |
| **Вервольф** | Человек | [Волк], всё надето (`installAllBeast`) | **2** (потолок игрока) + вечная `Rage` | конституция психики (`applyVitals=false`) | `WerewolfPsyche` |
| **Волк** | Волк | — | **0.45** | из органов | `WolfPsyche` |

Формула вервольфа: *игрок-на-максимуме + вечная ярость*. Формула волка: *органы вида × Э природной особи*.

## Рецепты

### Добавить орган виду
1. Числа — в `SpeciesBootstrap.CreateDefaults` (массив органов вида) или руками в инспекторе ассета.
2. Если орган включает способность — булев флаг `enablesX` в `Organ` (`SpeciesSO.cs`), провод в `CreatureBody.Recompute` (по образцу `enablesKick`), компонент способности на игроке.
3. **Прогнать `Chimera → Создать дефолтные виды` + Ctrl+S** (ассеты сами не обновляются!).

### Добавить новый вид
1. Ассет вида: по образцу в `SpeciesBootstrap` (или `Create → Chimera → Вид (SpeciesSO)`). Органы = сразу и statblock NPC, и аугументы для игрока.
2. В доноры игрока вид попадает через `donors[]` тела игрока (бутстрап привязывает сам).
3. NPC-префаб: `CharacterController + Health + Knockback + Stagger + HitFlash + Telegraph + NavLocomotion + Rage + доставки` + **`CreatureBody`** (chassis = вид, `expression` <1 для рядовой особи) + `SpawnVariance` (разброс особей).
4. **Своя психика** (тонкая!): решения по дистанции/состоянию, приёмы — через `TryUse`/`Tick` доставок; реализует `IBodyStatConsumer`, чтобы тело кормило урон/скорость из органов.

### Добавить способность-доставку
- **Для ИИ**: наследник `WindupAbility` — переопределить `TelegraphColor`, `OnTick()` (замах/исполнение/правила отмены), при нужде `Abort(hard)` (пример «закоммиченного» приёма — полёт `LeapAbility`). Психика зовёт `TryUse()` и тикает `Tick()`, кулдаун атак держит психика (это её ритм-решение).
- **Для игрока**: `MonoBehaviour + IAbility.TryUse()` (кулдаун внутри), бинд в `PlayerInputDriver`, при органной привязке — флаг `enablesX` (образец: `PlayerHowl`).

### Добавить эффект удара
Одна ветка: значение в `EffectKind`, фабрика в `HitEffect`, `case` в `Hit.Apply`. Проверено на `Stun`.
Кандидаты в очереди (ждут носителей): отравление, кровотечение (нужен DoT-тикер в `Health`), страх (дёрнет `WolfPsyche.AddFear`).

### Сделать химеру-NPC (мини-босс)
Префаб с `CreatureBody`: `chassis = X`, `donors = [Y]`, `installAllBeast = true`, `expression = N`, `applyVitals` по вкусу (false — витальность задаёт психика) + своя психика. Пример-эталон: генератор `Chimera → Создать префаб Вервольфа` (`WerewolfPrefab.cs`).

## Тюнинг-карта (что где крутить)

| Что | Где |
|---|---|
| Числа органов, пул, цены | ассеты `Data/Человек.asset`, `Data/Волк.asset` (правишь бутстрап → перегоняешь меню) |
| Сила особи NPC | `Expression` на `CreatureBody` префаба |
| Разброс особей | `SpawnVariance` (±HP/урон/скорость) |
| Ярость | `Rage` на префабе (множители, `permanent`); длительность от воя — `howlRageDuration` (волк) / `rageDuration` (босс) |
| Стая: жетоны, мораль, паника | объект `PackCoordinator` в сцене |
| Приёмы (урон/дальности/замахи) | компоненты `BiteAbility`/`LeapAbility` на префабах |
| Родство: скидка и кривая мощи | `CreatureBody` игрока (`discountPerAffinity`, `bonusStart/Full`, `maxBonusMult`) |
| Конструктор-UI | `ConstructorUI` (Tab, замедление ×10 — `timeScaleWhenOpen`) |

## Гочи (Unity и наши)

- Изменил данные в `SpeciesBootstrap` → **обязательно** `Chimera → Создать дефолтные виды` + Ctrl+S.
- Изменил генератор босса → **перегенерируй** `Chimera → Создать префаб Вервольфа` (GUID префаба сохраняется, ссылки живы).
- `RequireComponent` до-создаёт компоненты в рантайме (лог «Creating missing…») — чтобы убрать и тюнить, открой префаб + Ctrl+S.
- Ассеты двигать/переименовывать **только с `.meta`** (ссылки по GUID). Скрипту — rename файла+класса+`.meta` разом.
- Переименовал сериализованное поле — `[FormerlySerializedAs("старое")]`, иначе префабы потеряют значение.
- `UnityEvent` при `AddComponent` в рантайме — инициализировать `= new()`.
- Перф-правила: один `Update`-тик на существо (способности тикает психика), ноль аллокаций на удар (эффекты — value-типы).
