# CHIMERA — Конструктор существ: инструкция

> Как устроена система «шасси + аугументы» и как через неё добавлять виды, органы, способности и эффекты.
> Актуальна после рефактора существ (спека: `Docs/superpowers/specs/2026-07-01-creature-abstraction-design.md`).

## Большая картина — три закона

1. **ТЕЛО строится единообразно** у всех существ: шасси + органы (аугументы) → статы + способности + восприятие. Тело — это **данные** (`SpeciesSO` + `CreatureBody`).
2. **ПСИХИКА пишется индивидуально** под вид — это **код**, но тонкий: только решения (sense → think → act) поверх «глаголов тела». Психика не переизобретает навигацию/телеграф/приёмы. Ориентир-словарь режимов: **СПОКОЙСТВИЕ / НАСТОРОЖЕННОСТЬ / АТАКА** — вид маппит их по-своему (млекопитающие — полный спектр + мораль/ярость; змея — «холодный расчёт» без паники: засада → оценка добычи → бросок).
3. **СПОСОБНОСТЬ = ДОСТАВКА + ЭФФЕКТЫ.** Доставки (укус-конус, прыжок…) переиспользуются между существами с разными числами; эффекты — общий словарь, ортогональный доставкам.

Фикция = архитектура: **способность живёт на органе**, потому что химеризация = кража органа. Числа органов записаны в **мутагенной шкале** — насколько они раскрыты в конкретном теле, решает **экспрессия**.

**Слот против аугумента (принцип, проработать при смене шасси):** СЛОТ — атрибут ШАССИ (у змеи есть «Тело», у человека нет). Органы вида = его **естественные аугументы**, заполняющие его же слоты; в чужом теле тот же орган — аугумент-трансплант (нейминг с точки зрения получателя: у шасси змеи ходовая часть «Тело-хвост», аугумент игроку — «Змеиный хвост»). **Ходовые части шасси помечаются `Organ.chassisOnly`** — аугументом не крадутся (закон «локомоция = свойство шасси»), в доноры/химерный слот не попадают. Мэппинг слотов между разными шасси — НЕ проработан, ждёт систему смены шасси (post-MVP).

**Слоты (канон код↔GDD).** База легконогого шасси (человек/волк/лось): **Руки · Ноги · Пасть · Чутьё · Сердце · Шкура**. Безногое шасси (змея) меняет Ноги на **`Тело`** (ходовая, `chassisOnly`) и не имеет Рук. **`Хвост`/придаток** — АДДИТИВНЫЙ слот (дорастает; игроку — химерный слот): змеиный хвост, лосиные рога — по видовому имени (обобщать в «Придаток» не раньше 3-го примера). Расхождения с GDD-таблицей слотов (это уточнения, не противоречия): GDD зовёт голову ОДНИМ слотом («голова»/«морда»), а код разнёс её на **Пасть** (атака-рот: укус/вой/рёв) + **Чутьё** (чувства) — держим РАЗДЕЛЬНО ради микса (термо-глаза змеи + укус волка на одной голове; сам GDD трактует Пасть как отдельный атакующий слот). «Торс» GDD = слот **Сердце** в коде.

## Словарь

| Термин | Что это | Где живёт |
|---|---|---|
| **Вид** | Ассет: имя, тинт, пул мутагена, органы. Две роли: ШАССИ (слоты + дефолт-органы) и ДОНОР (источник аугументов) | `SpeciesSO`, ассеты в `Assets/_Chimera/Data/` |
| **Орган** | Вклад в статы своего слота + флаги способностей (`enablesBite/Scent/Kick/Howl`) + цена в пуле | класс `Organ` внутри `SpeciesSO.cs` |
| **Экспрессия** | Насколько раскрыты гены ВИДА в теле — ОДНОРОДНО для звериных органов И для органов ШАССИ (человек — полноценный вид). Игрок — авто от родства с видом органа, ЕДИНАЯ РАВНОМЕРНАЯ кривая 0→100: и скидка (0→−80%), и мощь (×1→×2) растут линейно (каждое убийство и дешевит, И усиливает — прежней «мёртвой зоны» силы 0–80 нет). На 100 Человек человеческие органы ×2 (база их ×0.75 → нетто ×1.5). Вервольф — фикс 2; природный волк — 0.45. Времена/дальности не масштабируются. **База бленда звериного органа — СЫРОЙ человеческий (не ×2), только чистый слот шасси экспрессится** | поле `expression` на `CreatureBody` |
| **Витальность** | **БАЗА ШАССИ × (1 + Σ бонусов органов × экспрессия) × разброс особи.** База (`SpeciesSO.baseHp`) — «тело как таковое», её экспрессия НЕ трогает; орган Сердца даёт ДОЛЮ базы (`Organ.hpBonus`, 0.5 = +50%), и раскрывается экспрессией уже она. Раньше HP были абсолютами и масштабировались целиком — «Э 0.45» значило «волк на 45% статов» (полудохлый), а числа органов подбирались, лишь бы это скомпенсировать. Процентная модель чинит и смысл экспрессии, и перенос между шасси (человеческое сердце в лосином теле больше не делает лося хилым) | `SpeciesSO.baseHp` + `Organ.hpBonus`, сборка — `CreatureBody.Recompute` |
| **Пул мутагена** | Ёмкость тела; каждый слот занимает пул; химеризация = размен в ёмкости | `SpeciesSO.mutagenPool` |
| **Тело** | Универсальный компонент сборки: у игрока динамика (Tab/хоткеи), у NPC — застывшая конфигурация | `CreatureBody` |
| **Доставка** | Как приём достигает цели. ИИ: `BiteAbility`, `LeapAbility` (замах через базу `WindupAbility`). Игрок: `PlayerAttack/Bite/Kick/Howl` (контракт `IAbility.TryUse`) | `Scripts/Combat/`, `Scripts/Player/` |
| **Эффект удара** | Что происходит при попадании: `Damage, LifeSteal, Knockback, RegenDebuff, Stun, Venom` | `Hit.cs` (enum + одна ветка на эффект) |
| **Статусы** | `Stagger` — два калибра: стаггер <1с (боёвка, от попаданий) и стан ≥1с (эффект). `Rage` — ярость: +урон +скорость, −защита, бесстрашие. `Venom` — накопительный яд (1 стак стоп-реген → 2 уязвимость → 3 DoT). `Grabbed` — ЕДИНЫЙ захват (хвост игрока / кольца змеи / будущий богомол): фиксация/укрепление бьёт импульс-стагером, дальше жертва действует ПО СИЛЕ хвата — слабый хват = стоишь, но бьёшь схватившего; защёлк (Locked) = стан (спека 2026-07-17). `Morale` — УНИВЕРСАЛЬНАЯ шкала духа (страх↔ярость, стаки ±1×10с; вешает `CreatureBody` любому NPC, психики читают): порог храбрости от личности, паника ниже −порога, коммит выше +порога (ярость), морда = градусник. Маркеры тела: `ColdBlooded` (невидим термо, иммунен менталу — Fear/внешняя Rage/**Morale**; источник — сердце змеи), `Camouflage` (невидим в неподвижности), `Massive` (резист нокбэка; захват — на стадию слабее → защёлк недостижим, всегда отбивается), `Satiety` (СЫТОСТЬ — ЕДИНАЯ механика «поел → бонус-реген»; вешает `CreatureBody.CreditKiller` на смерть жертвы, только ХИЩНИКУ `SpeciesSO.eatsMeat`: убийце полный, стае рядом половина, глотающему целиком `DigestsWhole`/змея — двойной; уроном НЕ прерывается; объединила бывшее «переваривание» — лечение в ядро, ПОВЕДЕНИЕ сытости на психике: змея-NPC прячется на насест по `IsSated`, ёж/волк — ничего особого), `Slow` (замедление стаками, иглы ежа — читает единая точка хода) | `Scripts/Combat/`, `Scripts/Senses/` |
| **Цвет-сигналы** | ЕДИНАЯ легенда, ТРИ раздела: **приёмы** (телеграф = СВОЁ действие: укус красный, прыжок оранжевый, захват фиолетовый, чардж розовый, вой сиреневый, рога зелёно-бирюзовый, меч стальной, пинок лаймовый), **статусы** (белёсый = стан/схвачен — красит `StunTint`) и **эмоции** (красят ГОЛОВУ, пока действуют: ярость — бордовая морда, паника — синяя; тело = приёмы, голова = эмоции — каналы разнесены пространственно; красит `EmotionTint` через головной рест-слой `Telegraph`, конвенция имён головных частей `Head/Muzzle/Nose/EarL/EarR`, психика может задавать градиентное настроение `SetMood` — лесенка лося). **Приоритет-стек** `Telegraph`: стан > вспышка приёма > эмоц-рест > натуральный (вспышки откатываются К РЕСТУ, `HitFlash` восстанавливает через `Reapply`). ТА ЖЕ легенда красит **отладочные гизмо-хитбоксы** приёмов (`WindupAbility.OnDrawGizmos` + гизмо игроцких приёмов). **Стадийные приёмы** (обхват змеи) телеграфят ГРАДИЕНТОМ от реста к цвету приёма (`Telegraph.SetGradient`). Реляционных цветов не заводим | `TelegraphColors.cs`, `StunTint.cs`, `EmotionTint.cs` |

## Кто на чём собран (четыре существа)

| | Шасси | Доноры | Экспрессия | Витальность | Водитель |
|---|---|---|---|---|---|
| **Игрок** | Человек | [Волк] | авто (родство) | из органов | `PlayerInputDriver` |
| **Вервольф** | Человек | [Волк], всё надето (`installAllBeast`) | **2** (потолок игрока) + вечная `Rage` | из органов (костыль `applyVitals=false` снят: человечья база × максимум волчьих % = те же 300 HP; «не танк» — свой `maxDamageReduction` 0.15 в данных тела) | `WerewolfPsyche` |
| **Волк** | Волк | — | **0.45** | из органов | `WolfPsyche` |
| **Змея** | Змея | — | **0.5** | из органов | `SnakePsyche` (соло-засадник: термо, яд, обхват, камуфляж+гремок) |

Формула вервольфа: *игрок-на-максимуме + вечная ярость*. Формула природной особи (волк/змея): *органы вида × Э*. **Родство — УБИЙЦЕ и ЛОКАЛЬНОЕ**: хранится в теле каждого существа (`CreatureBody.GetAffinity/AddAffinity`; глобального трекера нет); на смерть тело кредитует того, кто убил (`Health.LastAttacker`, ставят `Hit.Apply`/яд/удушение), +1 за уникальный видо-флаг жертвы — вервольф даёт убийце +1 Человек и +1 Волк; волк, задушенный змеёй, «достаётся змее» (задел эволюции химер-NPC). UI/dev/спавнеры читают тело игрока через `CreatureBody.PlayerBody`.

## Рецепты

### Добавить орган виду
1. Числа — в `SpeciesBootstrap.CreateDefaults` (массив органов вида) или руками в инспекторе ассета.
2. Если орган включает способность — булев флаг `enablesX` в `Organ` (`SpeciesSO.cs`), провод в `CreatureBody.Recompute` (по образцу `enablesKick`), компонент способности на игроке.
3. **Прогнать `Chimera → Создать дефолтные виды` + Ctrl+S** (ассеты сами не обновляются!).

### Добавить новый вид
1. Ассет вида: по образцу в `SpeciesBootstrap` (или `Create → Chimera → Вид (SpeciesSO)`). Органы = сразу и statblock NPC, и аугументы для игрока.
2. В доноры игрока вид попадает через `donors[]` тела игрока (бутстрап привязывает сам).
3. NPC-префаб: `CharacterController + Health + Knockback + Stagger + HitFlash + Telegraph + NavLocomotion + Rage + доставки` + **`CreatureBody`** (chassis = вид, `expression` <1 для рядовой особи) + `SpawnVariance` (разброс особей).
4. **Своя психика** (тонкая!): решения по дистанции/состоянию, приёмы — через `TryUse`/`Tick` доставок; реализует `IBodyStatConsumer`, чтобы тело кормило урон/скорость из органов.

### Добавить способность-доставку
- **Для ИИ**: наследник `WindupAbility` — переопределить `TelegraphColor`, `OnTick()` (замах/исполнение/правила отмены), при нужде `Abort(hard)` (пример «закоммиченного» приёма — полёт `LeapAbility`). Психика зовёт `TryUse()` и тикает `Tick()`, кулдаун атак держит психика (это её ритм-решение).
- **Для игрока**: `MonoBehaviour + IAbility.TryUse()` (кулдаун внутри), бинд в `PlayerInputDriver`, при органной привязке — флаг `enablesX` (образец: `PlayerHowl`).

### Добавить эффект удара
Одна ветка: значение в `EffectKind`, фабрика в `HitEffect`, `case` в `Hit.Apply`. Проверено на `Stun`, `Venom`, `Bleed`, `Slow`.
`Slow` (иглы ежа) — накопительное замедление: стаки тают, читаются ЕДИНОЙ точкой хода (`NavLocomotion.Move` у NPC, `PlayerController` у игрока), глубина = число попавших снарядов.
Кандидаты в очереди (ждут носителей): страх (дёрнет `WolfPsyche.AddFear`).

### Добавить новую сенсорику (ось восприятия)

Новое чувство = **новая ОСЬ на всех телах**, у оси две стороны: СЕНСОР (кто чует) и ЭМИТТЕР (кого чуют). Эталоны: запах (полная ось), термо (ось с исключением). Чек-лист:

1. **Назови атрибут оси** и выбери дефолт так, чтобы существующие виды НЕ требовали правок («тёплый по умолчанию», «шумный по умолчанию»). Исключение из оси = компонент-маркер (`ColdBlooded`). Если атрибут зависит от ПОВЕДЕНИЯ (шум — от движения/действий), эмиттеру нужны события, а не маркер — это самый дорогой вариант, закладывай его в оценку.
2. **Сенсорная сторона**: статик-запрос в `Perception` (`Sees` — ЗРЕНИЕ целиком, `HasLineOfSight`, `SeesThermal`; для игрока — статик-тоггл по образцу `WolfScent`). Психика вида своё чувство ПРЕДПОЛАГАЕТ (радиус — её поле); флаги органов гейтят только игрока. **Правило канала пишется ОДИН раз в `Perception`, психика его только зовёт** — зрение уже разъезжалось по трём копиям (лось терял камуфляж).
3. **Эмиттерная сторона**: компонент на телах — сам в Awake психики (образец `ScentTrail`) или от флага органа через `CreatureBody` (образец `ColdBlooded`/`Camouflage`).
4. **Флаги органов**: кто ДАЁТ чувство (`enablesScent`) и кто даёт ИСКЛЮЧЕНИЕ (`coldBlooded`, `camo`); провод в `CreatureBody.Recompute`; статики игрока — строго под гвардом `move != null` (тело NPC не должно включать чувство игроку!).
5. **RPS-проверка**: что ось контрит и что контрит её. У каждой скрытности ≥1 зацепка (невидимую змею выдают гремок + запах).
6. **Симметрия игрока**: украл орган → получил чувство И/ИЛИ исключение — тело раздаёт само по флагам.
7. **Дебаг-видимость**: как ось увидеть до настоящего UI — визуал (след/проблеск/силуэт), строка в DebugHud, дев-тоггл.
8. **Носители (YAGNI)**: ось строится только вместе с видом-носителем, против которого ей играть.

Пример на будущее — сова/эхолокация: ось «шумность», все шумят при движении (дефолт, но нужны СОБЫТИЯ шума — см. п.1), исключение «бесшумный», сенсор слышит сквозь темноту; контрится неподвижностью/тишиной.

### Скрытность: камуфляж, прокрадывание, переноска (эталоны из змеи)

- **Камуфляж (`Camouflage`, флаг Шкуры):** прячет меши ТЕЛА, пока существо (а) почти неподвижно (скорость — по смещению `transform`, работает и для `CC.Move`, и для transform-климба) И (б) не РАСКРЫТО. Раскрытие = телеграф (замах/приём/захват/гремок) ИЛИ боль (стаггер/обхват) ИЛИ **полученный урон** (`damageRevealTime` — «нельзя прятаться, пока рвут»: закрывает эксплойт инвиза под рвущей стаёй). Каналы чувств (термо-контур, аура запаха, погремушка) камуфляж НЕ трогает — их прячут свои системы.
- **Стелс на ходу (`HoldStealth`):** психика держит невидимость В ДВИЖЕНИИ (крадущийся охотник-невидимка не собирает рой), но раскрытие перебивает. Змея зовёт его на прокрадывании, НЕ на бегстве (бежит видимой — стая должна гнать).
- **Прокрадывание против осцилляции (`Prowl`):** неподвижные скрытные приманки сгоняют стаю в осциллирующие кучи между собой. Праздный засадник должен ДВИГАТЬСЯ (к одинокой тёплой жизни / прочь от толпы / бродить). Урок общий: стационарный аттрактор = осцилляция.
- **Несомая добыча (`ICarried`):** хищник может НЕСТИ жертву (змея тащит волка на насест). Носимая — кукла: своя локомоция/гравитация выключены (`SetCarried`), позицией владеет носитель (иначе стан тянет тело вниз гравитацией). Общий контракт под будущую тёплую добычу (YAGNI: пока один носитель — волк).

### Сделать химеру-NPC (мини-босс)
Префаб с `CreatureBody`: `chassis = X`, `donors = [Y]`, `installAllBeast = true`, `expression = N`, `applyVitals` по вкусу (false — витальность задаёт психика) + своя психика. Пример-эталон: генератор `Chimera → Создать префаб Вервольфа` (`WerewolfPrefab.cs`).

## Тюнинг-карта (что где крутить)

| Что | Где |
|---|---|

## Гочи (Unity и наши)

- Изменил данные в `SpeciesBootstrap` → **обязательно** `Chimera → Создать дефолтные виды` + Ctrl+S.
- Изменил генератор босса → **перегенерируй** `Chimera → Создать префаб Вервольфа` (GUID префаба сохраняется, ссылки живы).
- `RequireComponent` до-создаёт компоненты в рантайме (лог «Creating missing…») — чтобы убрать и тюнить, открой префаб + Ctrl+S.
- Ассеты двигать/переименовывать **только с `.meta`** (ссылки по GUID). Скрипту — rename файла+класса+`.meta` разом.
- Переименовал сериализованное поле — `[FormerlySerializedAs("старое")]`, иначе префабы потеряют значение.
- `UnityEvent` при `AddComponent` в рантайме — инициализировать `= new()`.
- Перф-правила: один `Update`-тик на существо (способности тикает психика), ноль аллокаций на удар (эффекты — value-типы).
