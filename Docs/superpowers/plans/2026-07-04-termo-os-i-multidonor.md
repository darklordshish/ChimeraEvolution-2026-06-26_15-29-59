# План: доделка термо-оси + мультидонор (черновик к слайсу 2)

> ЧЕРНОВИК. Происхождение: baseline-тест «спланируй доделку термо» (агент-архитектор, read-only).
> План оценён на 8/8 по чек-листу оси восприятия (`CONSTRUCTOR_GUIDE.md → Добавить новую сенсорику`),
> но НЕ утверждён пользователем и не сведён с мета/микс-слайсом (химерный слот + супремум + SuperBossReward).
> Перед исполнением — свести в единую спеку слайса 2.

## Разрыв (почему термо «недо-ось»)

Запах стоит на четырёх ногах: данные для ИИ (ScentField/Emitter/Trail) + игрок-как-сенсор через орган
(`enablesScent` → `Perception.WolfScent` → визуал) + RPS (камуфляж не прячет след) + отладка (HUD, тогглы).
Термо имеет только «ногу ИИ» (`Perception.SeesThermal` у змеи) и половину контр-механики (`ColdBlooded`):

- **Пит-орган — пустышка** (`SpeciesBootstrap`: cost 3, ни флага, ни статов); флага `enablesThermal` не существует.
- Игрок-как-термо-сенсор отсутствует: ни статики, ни визуала, ни HUD.
- `ColdBlooded` игроку недостижим: доноры игрока = [Волк], и `BuildSlots` берёт ПЕРВОГО донора на слот
  (волчий Нюх закрывает «Чутьё» — Пит-орган не встанет, пока нет выбора донора на слот).

## Срез 1 — термо-сенсор игрока (тест через дев-тоггл)

1. `Organ`: + `bool enablesThermal`, `float thermalRange` (радиус = данные органа; у змеи прецедент 14).
2. `Perception`: + `SnakeThermal`/`ThermalRange` (по образцу `WolfScent`) + хелпер `IsWarm(t)` (= есть `Health` И нет
   `ColdBlooded`) — единый источник правды «кто тёплый», переписать `SeesThermal` через него.
3. `CreatureBody.Recompute`: собрать `thermalOn`/`thermalRange` (radius суммировать без бленда — фикс-фича, как range);
   статики — строго под гвардом игрока `move != null`.
4. НОВЫЙ `Senses/HeatSignature.cs` (аналог ScentTrail): тёплый силуэт существа, если игрок-термо включён, цель в радиусе
   и НЕ ColdBlooded (проверять каждый Update — маркер меняется при смене Сердца). Рендер СКВОЗЬ стены → нужен материал/шейдер
   с `ZTest Always` (Sprites/Default глубину уважает — не подходит). Развилка визуала (решает пользователь):
   (а) дубли мешей-контуров — рекомендация, (б) аура-сфера как у следа.
   Гоча: шейдер только через `Shader.Find` вырезается из билда — материал делать ассетом.
5. Вешание: `HeatSignature` в Awake психик волка/вервольфа/змеи (по образцу ScentTrail; змея гасится сама по ColdBlooded).
   Префабы пересобирать не нужно.
6. `SpeciesBootstrap`: Пит-орган = `enablesThermal, thermalRange 14` + хоткеи змеиным органам (сейчас дефолт «1», конфликт).
7. Отладка: дев-тоггл T (форс термо) + строка в DebugHud.

Проверка: T → волки/босс светятся сквозь стены в радиусе; змея — нет; камуфляжную змею по-прежнему выдают гремок+запах.

## Срез 2 — честная достижимость: Змея-донор + выбор органа на слот

Ядро мета/микс-слайса. Сначала мини-спека (свести с химерным слотом и супремумом!).

1. `CreatureBody`: `Slot.beast` → СПИСОК альтернатив (все доноры по слоту, без `break`); `Toggle` = цикл
   человек→волк→змея→человек; цена/скидка — от конкретного варианта (`donorSpecies` пер-вариант, скидку даёт родство
   ИМЕННО этого вида); `GrantAffinityOnDeath`/`Recompute`/`SlotView` — адаптировать к списку.
2. `SpeciesBootstrap`: доноры игрока = [Волк, Змея]. Змеиное «Тело-хвост» на человеке не замапится (нет слота) — ок
   до слота «Хвост».
3. `ConstructorUI`/`PlayerInputDriver`: строка слота показывает цикл вариантов; хоткеи циклируют автоматически.

Проверка: Чутьё циклирует Нюх/Пит-орган (запах ИЛИ термо — билд-выбор); со змеиным Сердцем змеи игрока в засаде
НЕ замечают (SeesThermal=false). RPS замкнулся на живых билдах.

## Срез 3 — полировка

Баланс цен (Пит-орган 3, Сердце 5 — теперь антизмеиный стелс), CONSTRUCTOR_GUIDE (словарь органов, раскладка),
коммиты тематически раздельно, плейтест перед каждым.

## Осознанно НЕ делаем

- Гейт термо змеи от её органа (психика вида предполагает своё чувство — конвенция).
- NPC-vs-NPC восприятие (запах тоже только на игрока) — отдельный будущий разговор.
- Полноэкранная термо-камера/пост-эффект — GDD 5.5 относит к v1.0+; HeatSignature и есть «накладочный слой».
- Отдельный термо-эмиттер: тепло = жизнь (`IsWarm`), эмиттер без второго носителя — YAGNI.
