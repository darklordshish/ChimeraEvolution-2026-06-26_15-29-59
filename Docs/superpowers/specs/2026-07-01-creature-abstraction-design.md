# CHIMERA — Абстракция существ: тело/психика, способности как доставка+эффект

- **Дата:** 2026-07-01
- **Статус:** дизайн одобрен (секции 1–3); ждём ревью спеки → план по фазам.

## Проблема

Набрана критическая масса механик (бой, восприятие/запах, психика стаи, вид + босс-химера, органы-аугументы). Существа реализованы монолитами: `WolfAI` и `WerewolfBoss` **независимо** переизобретают навигацию по NavMesh (`NavDir/ComputeNavDir/WanderTarget`), телеграф материала (`SetTelegraph`+`mpb`), восприятие и стейт-машину приёмов; игрок — отдельный стек. Дублирование растёт; новый вид добавлять дорого и неоднообразно. Цель — вытащить базис на **трёх существующих** примерах (Волк, Вервольф, Игрок), пока издержки минимальны.

## Три существа — одна ось

| Существо | Шасси | Сборка | Водитель |
|---|---|---|---|
| Волк | зверь | статичная | ИИ-психика |
| Вервольф | человек + волк (химера в ярости) | статичная (застывший лоадаут) | ИИ-психика |
| Игрок | человек + … | динамическая (конструктор) | ввод |

Разница — только в двух ручках: **сборка** (статичная/динамическая) и **водитель** (психика/ввод).

## Два закона

1. **Тело** строится единообразно: шасси + аугументы(органы) → статы + способности + восприятие. Это **данные**.
2. **Психика/мозг** пишется индивидуально под вид (ради богатой экосистемы). Это **код** — но психика **тонкая**: только решения поверх общих «глаголов тела», не переписывает навигацию/восприятие/телеграф.

Фикция = архитектура: способность живёт на органе, потому что химеризация = кража органа = кража его способности.

## Слои

- **`CreatureBody`** (общий, из данных): собран из шасси+органов. Владеет: композированными статами (+ разброс на спавне у мобов), набором способностей-доставок, профилем восприятия, актуатором движения. Глаголы: `Perceive()`, `Move(dir)`/`MoveTo(dest)`, `TryUse(ability)`.
- **Способность = доставка + эффекты**: компонент-доставка (`Bite`/`Leap`/`Grab`/`Howl`/`Melee`/`Kick`/`Ranged`), параметры из органа; на попадании emit `Hit` со списком эффектов из данных.
- **Эффекты** — маленький словарь, только **уже существующие**: `Damage`, `LifeSteal`, `Knockback`, `Stagger`, `Grab`, `RegenDebuff`. (`Venom`/DoT — первый новый, позже, как лакмус здоровья абстракции.)
- **`Driver`**: `PlayerInputDriver` (ввод → `TryUse`) **или** `Psyche` вида (sense→think→act). Психики: `WolfPsyche` (стая/мораль/страх/зов), `WerewolfPsyche` (ярость).
- **Сервисы (остаются как есть):** `Perception`, `ScentField/Emitter/Trail`, `PackCoordinator` (соц-сервис для стайных психик), `Health/Knockback/Stagger/HitFlash` (под телом). **Новые извлечения из дублей:** `Telegraph`, `NavLocomotion`.

## Поток кадра (sense → think → act)

- **NPC:** Психика → `body.Perceive()` + состояние стаи (**sense**) → решает цель / приём по дистанции / бегство по морали (**think**) → `body.Move()` + `body.TryUse(ability)` (**act**). Способность: замах + `Telegraph` → на попадании `Hit` со списком эффектов → эффекты применяются.
- **Игрок:** та же форма, `PlayerInputDriver` вместо психики. То же тело, те же приёмы, те же эффекты.

## Данные

`SpeciesSO` (шасси + органы), `Organ` (слот, статы + разброс, ссылки на доставки, профиль восприятия). Пресет-существо = шасси + лоадаут (+ для NPC — какая психика).

## Разброс на индивидах

Значимые статы (хп, урон) несут `±` в данных; при спавне моба катаются один раз (напр. волк 30±5 → 25/28/33). Игрок — детерминированная база (растёт от родства), разброс = 0. Один хук в сборке тела; психику и способности не трогает.

## Путь миграции (играбельно на каждой фазе)

- **Ф0 — бесплатные победы (поведение НЕ меняется):** извлечь `Telegraph` и `NavLocomotion` из дублей; `WolfAI`/`WerewolfBoss` зовут их. Куча тоньше, риск ноль.
- **Ф1 — словарь эффектов** (`Hit`/`HitEffect`): существующие атаки переводим на него, поведение то же.
- **Ф2 — доставки-способности как компоненты; мигрируем ИГРОКА первым** (его приёмы уже компоненты → легче ловить регресс, просто играешь) + `PlayerInputDriver`.
- **Ф3 — `CreatureBody`** (генерализация `ChimeraBody`); игрок живёт на нём.
- **Ф4 — волк** → `WolfPsyche` над `CreatureBody` + общие Nav/Telegraph/доставки; монолит `WolfAI` уходит. Проверка: стая/мораль/вой как настроены.
- **Ф5 — вервольф** → `WerewolfPsyche` (ярость); монолит `WerewolfBoss` уходит (переиспользует всё волчье-извлечённое → дёшев).
- **Ф6 — хук разброса на спавне** + чистка данных (`SpeciesSO`: ссылки на доставки, разброс, профиль восприятия). Вид №4 = данные + тонкая психика.

Между фазами — стоп / проверка / откат. Каждая фаза поведение-сохраняющая или аддитивная.

## Проверка

Автотестов в проекте нет — верификация **игрой**: запустить сцену, глазами свериться с целевым поведением фазы (`DebugHud` как наблюдатель). Правило: неожиданно поменялось наблюдаемое поведение — стоп и чинить до следующей фазы.

## Производительность

На масштабе игры (десятки существ + босс) оверхед схемы — в пределах шума; узкое место не здесь. Два дешёвых правила держат композицию бесплатной:

1. **Один `Update`-тик на существо.** Тикает водитель (психика/ввод); способности и эффекты — обычные вызовы методов, без собственных `Update`. Активная многокадровая способность тикается из тика существа. (Избегаем классического оверхеда «много `Update()`».)
2. **Ноль аллокаций на удар.** Эффекты собраны заранее (данные на органе/способности); `Hit` — struct или пул; применение идёт по заранее построенному списку. Уже практикуем эту дисциплину (переиспользуем `Collider[]`-буфер, `MaterialPropertyBlock`).

Реальная дорогая часть — **восприятие** (LOS-рейкасты, запросы к `ScentField`) — рефактором не меняется; централизация даже упрощает будущий троттлинг (обновлять не каждый кадр у каждого). Динамическая химеризация игрока (Add/Destroy компонентов) — на смене органа (редко, в конструкторе, время замедлено), не в кадре.

Если игра дорастёт до сотен существ — тогда data-oriented/ECS; сейчас YAGNI, и разделение тело(данные)↔психика(логика) как раз облегчит такой переход.

## Границы (YAGNI) / припаркованное

**НЕ строим сейчас** (крючки, которые схема делает дешёвыми): стамина, орудия/окружение, вертикаль, речь / соц-слой людей, мульти-сенс и цветовой спектр, `Venom`/DoT, вид №4.

**Условие окупаемости** (из честного разбора): сразу после рефактора — второй, **непохожий** зверь. Иначе получаем «красивый ткацкий станок с одной нитью».
