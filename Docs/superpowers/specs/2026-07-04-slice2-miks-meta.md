# Слайс 2 — МИКС и МЕТА: мультидонор, термо игрока, химерный слот, награда суперхимеры, хвост-нага

> Спека утверждена по 4 развилкам (пользователь): термо-визуал = СИЛУЭТЫ-ДУБЛИ; яд на игроцком укусе = ДА;
> хвост/нага = В ЭТОМ СЛАЙСЕ, ПОД КОНЕЦ (не icebox); награда вервольфа = +4 ПУЛА + ХИМЕРНЫЙ СЛОТ.
> Влитый черновик: `Docs/superpowers/plans/2026-07-04-termo-os-i-multidonor.md`. База: GDD §10, CONSTRUCTOR_GUIDE.

## Цель

Душа игры — микс РАЗНЫХ видов. Слайс делает микс реальным: игрок впервые собирает волка И змею в одном теле,
химерный слот даёт аддитивность (второе сердце), супремум делает дубли честными, вервольф становится
настоящей суперхимерой с разовой наградой. Финал — змеиный хвост с обхватом как доп-часть тела.

## Залоченные принципы

- **Выбор донора на слот:** слот предлагает все органы-альтернативы всех доноров (цикл человек→волк→змея→человек).
  Скидку на орган даёт родство ИМЕННО его вида.
- **Урон принадлежит доставке своего слота (игрок):** damage Рук → `PlayerAttack` (меч/коготь), damage Пасти →
  `PlayerBite` (укус). Сейчас весь damage суммируется в меч — со змеиными клыками (24) это сломалось бы.
  У NPC как было (психика получает сумму — их единственный мили и есть укус).
- **Яд игрока:** поле `venomStacks` на органе → `PlayerBite` вешает стаки (`HitEffect.Venom`). Пасть = развилка
  билда: волчья (вампиризм+вой) ↔ змеиная (яд). NPC-змея уже так работает.
- **Химерный слот:** универсальный аддитивный слот сверх шасси. Принимает ЛЮБОЙ донорский орган; цена ×2
  (`chimeraSlotMult`, тюнится) поверх обычной формулы. Первый слот — награда вервольфа; лимит пока 1.
- **СУПРЕМУМ (агрегация дублей):** вклады в один стат от органов ОДНОГО типа слота (нативный + химерный дубль
  того же `organ.slot`) складываются НЕ суммой: скаляр → `max`, флаг → `OR`. Разные типы слотов — сумма, как
  сейчас (сегодня каждая группа = 1 орган, поэтому поведение не меняется до первого дубля). Второе сердце ≠ ×2
  реген; окупается НОВЫМ направлением (холоднокровность).
- **Орган без слота у шасси** (Хвост, Тело) в химерном слоте образует НОВУЮ группу (идёт суммой) — это
  «заработанный дом для не-нативных органов» из GDD. Баланс-риск «Тело-хвост = +скорость за 10 очков» —
  смотреть игрой, цена ×2 сама давит.
- **`SuperBossReward`:** компонент на префабе суперхимеры. ПЕРВОЕ убийство типа (`typeId`, статик-реестр в
  сессии — как `AffinityTracker`, до мета-сейва) → +4 к пулу + анлок химерного слота. Повторные — только
  обычное родство (флаг-скан тела: +1 Человек, +1 Волк).
- **Хвост — слот-оружие** (не локомоция): орган «Удушающий хвост» (slot «Хвост») ставится в химерный слот →
  нага. Обхват игроком NPC — фирменная способность; детальный дизайн — сессия с пользователем на подходе к 2e.

## Под-слайсы (играбельно после каждого)

### 2a — Мультидонор + расщепление урона + яд игрока
- `CreatureBody`: `Slot.beast` → список вариантов `(Organ, вид)` — `BuildSlots` собирает ВСЕ совпадения по слоту
  (убрать `break`); `Toggle` = цикл; цена/скидка/`donorSpecies` — пер-вариант; `SlotView` показывает текущий и
  следующий; `GrantAffinityOnDeath`/`Recompute` — на список.
- `Recompute` (игрок): damage Рук → `attack.SetMelee`, damage Пасти → `bite.SetDamage`; `venomStacks` → `bite`.
- `SpeciesSO.Organ`: + `int venomStacks`. Бутстрап: Ядовитые клыки `venomStacks = 1`, хоткеи змеиным органам
  (Пасть 5, Сердце 3, Шкура 6, Чутьё 4 — дефолтная «1» конфликтует).
- Бутстрап: доноры игрока = `[Волк, Змея]`. `ConstructorUI`/`PlayerInputDriver`: строка слота с циклом.
- **Проверка:** Пасть циклирует Рот/Волчья(вампиризм+вой)/Клыки(яд — стаки видны на волке в HUD? нет, на цели);
  Сердце циклирует до Хладнокровного (200 HP + реген + невидим для термо змей — змея в засаде не атакует);
  Шкура до Чешуи (броня + камуфляж в неподвижности у игрока!). Меч НЕ усиливается от смены Пасти.

### 2b — Термо-ось игрока (по рецепту «ось восприятия» из CONSTRUCTOR_GUIDE)
- `Organ`: + `enablesThermal`, `thermalRange`. Пит-орган: `enablesThermal, thermalRange 14`.
- `Perception`: + `SnakeThermal`/`ThermalRange` (образец `WolfScent`) + хелпер `IsWarm` (Health && !ColdBlooded),
  `SeesThermal` через него. Провод в `Recompute` под гвардом игрока (`move != null`).
- НОВЫЙ `Senses/HeatSignature.cs`: СИЛУЭТЫ-ДУБЛИ — дубли мешей тела с тёплым glow-материалом, `ZTest Always`
  (сквозь стены); материал — ассет (гоча: `Shader.Find`-only вырезается из билда). Гаснет: не в радиусе / термо
  выключено / цель `ColdBlooded` (проверять каждый кадр — Сердце меняется в рантайме). Вешается в Awake психик
  (волк/вервольф/змея; змея гасится сама — холоднокровна).
- Отладка: дев-тоггл `T` (форс-термо), строка в DebugHud.
- **Проверка:** Чутьё = Пит-орган → тёплые контуры волков/босса сквозь стены в радиусе; змея не светится;
  Нюх ↔ Пит-орган = билд-выбор «запах ↔ тепло» (RPS в руках игрока).

### 2c — Химерный слот + супремум (гейт: dev-грант)
- `CreatureBody`: химерный слот в данных (пустой доп-слот: human=null, принимает любой донорский орган),
  `chimeraSlots` (int, пока 0/1), цена = формула × `chimeraSlotMult` (2). UI: строка «Химерный [7]» (хоткей 7),
  цикл по ВСЕМ органам доноров.
- `Recompute` → супремум: вклады группируются по `organ.slot`; внутри группы скаляры max, флаги OR; группы
  суммируются. (Регресс-проверка: без дублей числа НЕ меняются.)
- Dev-панель: кнопка «+химерный слот» (леса до 2d).
- **Проверка:** второе сердце (волчье+змеиное): HP = max(150,200)=200, реген = max(2,4)=4, холоднокровность
  есть, вой есть — НЕ 350 HP/6 регена. Пул трещит (×2) — выбор больно-честный.

### 2d — `SuperBossReward` на вервольфе
- Компонент: `typeId="Вервольф"`, `poolBonus=4`, `grantsChimeraSlot=true`. На `onDeath`: если тип НЕ в
  статик-реестре сессии → `CreatureBody.ExpandPool(4)` игроку + `chimeraSlots++` + реестр. Генератор
  `WerewolfPrefab` вешает. HUD-строка о полученной награде (простая).
- **Проверка:** первое убийство босса → пул 16→20, слот открылся; второе → только родство. Петля «босс →
  качественный скачок сборки» играется.

### 2e — Хвост как ДОП-ЧАСТЬ ТЕЛА: обхват игрока (дизайн УТВЕРЖДЁН на сессии с пользователем)

> Терминология (поправка пользователя): это НЕ нага. Хвост в химерном слоте = дополнительная часть
> тела на человеческом шасси (ноги на месте). НАГА/кентавр = СПЛАВ ШАССИ (мифический body plan) —
> далёкое будущее, к этому слайсу не относится.

**Рамка: держишь ХВОСТОМ, не телом** → руки свободны (меч/укус работают — яд+контроль = синергия),
цена = ТВОЯ мобильность. Сила хвата зависит от носителя: аугумент-хвост (человеческое шасси, химерный
слот) — кап **ст.2**; змеиное шасси (хвост = тело, post-MVP) — кап ст.3 (удушение-DoT придёт там же).

- Орган Змеи «Удушающий хвост» (slot «Хвост» — у человека слота нет → живёт ТОЛЬКО в химерном слоте).
  Флаг `enablesConstrict` → `PlayerConstrict` (кнопка **F**; повторное F = отпустить).
- **Стадии (зеркало змеи, время работает НА игрока):** ст.1 — жертва обездвижена (продлеваемый стан),
  ты замедлен (~×0.55), жертва ВЫРЫВАЕТСЯ через случайное время (гонка «продержи до защёлка»);
  ст.2 (grip ≥ порога) — ЗАЩЁЛКНУТО: сама не вырвется, ты замедлен сильнее (~×0.3). DoT-удушения у
  аугумент-версии НЕТ — это инструмент контроля, убиваешь укусами/мечом.
- **Масса жертвы = −1 стадия:** маркер `Massive` (вервольф) → аугументом держится только ст.1
  (замедлил на секунды — уже ценно), со змеиным шасси — до ст.2. Иммунов нет — есть градация.
- **Срывы:** разовый урон по тебе ≥ порога (стая спасает своего — зеркало «бей змею»); ты ушёл
  дальше дистанции хвата (рывок автоматически рвёт); ты сам схвачен (волчий грэб); повторное F.
  Жертва при вырывании отлетает (Knockback), тебе — кулдаун обхвата.
- Схваченная/оглушённая жертва РАСКРЫТА для камуфляжа (боль выдаёт — обхваченная змея не исчезает).
- **Проверка:** человек с химерным хвостом держит волка (ст.1 дёргается-вырывается, ст.2 висит
  намертво), закусывает ядом; второй волк бьёт тебя — хватка слетает; вервольф стряхивает после ст.1.

## Числа (стартовые, тюнить игрой)

`chimeraSlotMult 2` · награда: +4 пула, +1 химерный слот, лимит 1 · Пит-орган: cost 3, range 14 ·
клыки: venomStacks 1 · реестр боссов и пул-бонус — в сессии (мета-сейва нет).

## Осознанно НЕ в этом слайсе

Мета-сейв (перманентность наград) · NPC-vs-NPC чувства · термо-пост-камера (v1.0+, HeatSignature = слой) ·
сеты/брейкпоинты · второй тип суперхимеры · выбор донора в UI-созвездиях (шаг 3 UI) · лазание по стенам.
