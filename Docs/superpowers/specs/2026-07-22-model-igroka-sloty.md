# Модель игрока: части по слотам органов

Дата: 2026-07-22. Продолжение `2026-07-22-modeli-blender-pipeline.md`.
Статус: дизайн согласован, генератор в работе.

## Зачем именно так

Химеризация — ядро игры: игрок меняет органы по слотам прямо в бою (хоткеи 1–7),
и GDD называет свап «органическим переморфингом». Разд. 5.5 прямо говорит, что
ручной тоггл 1/3 лица оставлен в MVP **чтобы наблюдать переморфинг тела**.

Отсюда главное требование: **граница слота = граница сменной детали**. Модель,
нарезанная «как удобно анатомически», не переживёт первый же свап.

## Что уже диктует код (контракт, не пожелания)

| Правило | Источник |
|---|---|
| Головные части: `Head`, `Muzzle`, `Nose`, `Jaw`, `Ear*` — их красят эмоции (ярость бордовая, паника синяя) | `Telegraph.IsHeadName` |
| `Head` и `Nose` прячутся от первого лица по имени | `PlayerController.SetFirstPerson` |
| Тинт состава красит все дочерние рендеры, лицо (глаза/брови/борода) исключено по именам | `CreatureBody.UpdateTint` |
| Камуфляж прячет меши тела | `Camouflage` |

## Слоты → детали

Канон слотов (CONSTRUCTOR_GUIDE): **Руки · Ноги · Пасть · Чутьё · Сердце ·
Шкура** + аддитивный придаток (**Хвост**, **Рога**).

Голова разнесена на два слота намеренно — чтобы собиралось «термо-глаза змеи +
укус волка на одной голове». Значит голова не может быть одной деталью.

| Слот | Детали | Заметки |
|---|---|---|
| **Пасть** | `Jaw`, зубы | сменная группа: человеческая челюсть ↔ волчья морда |
| **Чутьё** | `Nose`, `EyeL`/`EyeR`, `EarL`/`EarR` | глаза остаются человеческими дольше всего |
| **Руки** | плечо → предплечье → `HandL`/`HandR` | **кисть — точка свапа** на коготь |
| **Ноги** | бедро → голень → стопа | стопа ↔ лапа |
| **Шкура** | пока материал | накладки-шерсть — позже |
| **Сердце** | — | невизуален |
| **Придаток** | сокеты `Socket_Tail`, `Socket_Horns` | у человека пустые |

Лицо (`EyeL`, `EyeR`, `BrowL`, `BrowR`, `Beard`) — отдельные детали, исключены из
тинта. Борода учёного на озверевшем теле — намеренный образ, а не недосмотр.

## Кисть с пальцами — решено

Порядка 60 треугольников: большой палец отдельно, четыре слитых в блок. Причины:

1. От первого лица кисти ближе всего к камере.
2. Слот «Руки» — носитель главного трейдоффа игры (коготь мощнее, но забирает
   оружие). Подмена должна читаться мгновенно; варежка → варежка не сообщает ничего.
3. Человек — единственный, кто эффективно использует оружие. Рука должна
   убедительно держать меч.

Заодно это каркас под когти.

## Вервольф как проверка конструктора

`WerewolfPrefab` уже собран как `chassis = Человек`, `donors = [Волк]`,
`installAllBeast = true`, `expression = 2` — то есть **босс есть игрок на максимуме
химеризации**.

Следствие: если человеческая модель принимает полный волчий комплект, вервольф
перестаёт быть кубами без отдельной модели. Босс — боевое крещение конструктора
моделей, и заодно проверка, что слоты нарезаны правильно.

Отличия босса от игрока-химеры: экспрессия и **масштаб** (босс крупнее). Значит
части должны масштабироваться — у `AttachWolfHead` уже есть параметр `k`.

## Границы первого шага

Делаем: **человек целиком по слотам + один звериный свап на пробу** — волчья
голова в слот Пасть (она уже готова в `Wolf.fbx`).

Стык человек↔зверь — главный технический риск: пивоты, масштаб, согласование
шеи с чужим черепом. Проверяем его на одном примере, а не на шести сразу.

Не делаем сейчас:

- **Визуализацию экспрессии** (×1…×2). Носителя нет, YAGNI. Части
  масштабируемые, «коготь длиннее при высоком родстве» добавится числом.
- **Шерсть геометрией** для слота Шкура — пока материал.
- Остальные звериные варианты (лось, змея) — после того как стык проверен.

## Образ

Атлетичный, рост ~1.9 м, шарнирная кукла в стиле волка (low-poly, flat shading,
части с пивотами в суставах).

Решение пользователя — атлетичный. Отмечено при обсуждении: текущий кубовый
`PlayerModel` описан как «худой — до озверения», и на контрасте с финальной
химерой худоба работала бы сильнее. Выбор сделан осознанно.
