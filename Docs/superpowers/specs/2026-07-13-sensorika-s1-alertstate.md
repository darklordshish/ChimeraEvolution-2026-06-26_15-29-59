# Сенсорный слайс, S1 — AlertState + SensoryProfile (унификация психик)

> Первый под-слайс СЕНСОРНОЙ RPS (суть игры). Задача пользователя: «привести механики в порядок и
> УНИФИЦИРОВАТЬ, чтобы двигаться дальше» + **заложить дешёвое расширение** под будущие виды/чувства.
> Строим станком (brainstorm→спека→план→код с плейтест-чекпоинтами), НЕ латаем точечно.

## Место в декомпозиции (весь сенсорный слайс)

Сенсорная RPS — это программа из 4 под-слайсов, каждый = свой вертикальный срез:

| # | Под-слайс | Суть | Зависит от |
|---|---|---|---|
| **S1** | **AlertState + SensoryProfile** (ЭТА спека) | 3-состоянчатая машина восприятия + пер-видовой сенсорный профиль; Страх/Ярость → эффекты | — (бэкбон) |
| S2 | Коллективный нюх / эмиссия следа NPC | NPC роняют след → триангуляция; per-perceiver выдыхание; пробивает камуфляж | S1 |
| S3 | Кросс-видовой грэб | обобщить `IGrabber` на любую цель | слабо от S1 |
| S4 | Чужеродность-химеры | чуждость из состава тела → NPC-реакция (Страх/Ярость/притяжение); резолв «цены химеризации» GDD §12 | S1 |

## Цель и констрейнты S1

- **Унифицировать** волчью и змеиную психику под ОДИН каркас: 3 состояния восприятия + сенсорный профиль вида. Психика остаётся ТОНКОЙ (закон 2) — каркас общий, наполнение состояний вид задаёт сам.
- **Пер-состоянчатая сенсорика = новый рычаг:** состояние гейтит, насколько зверь чувствует (спокойного можно обойти).
- **Страх и Ярость → боевые эффекты** (словарь `Hit`/`HitEffect`), а не состояния. Холоднокровие = иммунитет к обоим + рациональность.
- **Жёсткий констрейнт: не потерять наигранный фил.** Волк и змея после миграции ведут себя не хуже — валидируется плейтестом на каждый вид.
- **Дешёвое расширение** (см. отдельный раздел): новый вид/чувство добавляется данными, без переписывания движка.

## Архитектура: два слоя (развели в брейншторме)

**Слой 1 — состояние ВОСПРИЯТИЯ (универсально, общий код).**
**Слой 2 — сенсорный ПРОФИЛЬ вида** (данные: как излучает + как воспринимает). Их разделение — ключ, иначе S1 распухает.

### AlertState — 3 состояния (компонент на КАЖДОМ существе)

Крупнозернистые состояния восприятия; тонкая тактика («как именно атакую») живёт в психике.

- **Спокойствие** — ничего не чую. Идл-поведение (психика: блуждание / засада). Чувства РАССЛАБЛЕНЫ → окно прошмыгнуть.
- **Настороженность** — зацепка перевалила порог (сенсорика/шум/след/гремок), но цель не подтверждена. Крадусь-проверяю. Чувства ОБОСТРЕНЫ.
- **Атака** — цель подтверждена в боевых условиях. Психика гонит СВОЮ тактику.

Переходы двигает перцепция, которую психика скармливает `AlertState`; потеря цели/зацепки на `T` секунд → шаг вниз (Атака→Настороженность→Спокойствие). `AlertState` держит текущее состояние и отдаёт психике; психика читает `alert.State` и действует. **Ярость/Паника — НЕ состояния** (это эффекты, ниже) → машина ровно 3-состоянчатая.

### SensoryProfile — профиль вида (ДАННЫЕ, матрица-способный)

Данные на теле вида (`SpeciesSO`/`CreatureBody`), две стороны:

- **Излучение:** что вид радиирует в сенсорном пространстве — тепло (есть/нет; уже `HeatSignature`/`ColdBlooded`), запах (сила + скорость выдыхания; уже `ScentTrail.SetStrength`), шум (позже, с совой). Эмиттерная сторона в S1 формализуется как поля профиля, богатеет в S2.
- **Восприятие:** по КАЖДОМУ чувству — `дальность` + `острота` (порог чувствительности) + **множители по состоянию** [Спок, Настор, Атака]. Вот полная матрица «чувство × состояние».

Запрос восприятия становится функцией: `воспринимает ли A цель B по чувству X` = f(излучение B по X, дальность+острота A по X, состояние A). Пер-состоянчатый множитель применяется к дальности/остроте.

**Наполнение — ЛЕНИВОЕ (обкатка на простом):** матрица-способная форма, но ЗНАЧЕНИЯ для волка/змеи простые (в основном равномерные множители ~0.6/1.2/фокус; острота — дефолтная). Богатые ячейки (острота, per-perceiver выдыхание) заводятся в S2 и с приходом ежа/совы — там окупаются (правило трёх). Иначе — «станок с одной нитью», нечем тюнить/валидировать.

### Страх и Ярость — боевые эффекты (не состояния)

Уходят в словарь `HitEffect` + статус-компоненты, рядом с Ядом/Станом. Модификаторы ПОВЕРХ любого состояния.

- **`Fear`** (новый компонент + `HitEffect.Fear`, НАКОПИТЕЛЬНЫЙ — как стеки яда): навешивается с величиной, копится. Перевалил личный **порог храбрости** (личность) → бросок дел + бегство (rout) на время → затухает. Богаче длительностного: стая ломается вразнобой, храбрые держатся. Источники: вой-испуг игрока (`Frighten`), гибель собрата рядом (`AddFear`), позже — чуждость-химеры (S4). **Унифицирует два нынешних костыля** (`Frighten`-таймер + `AddFear`-аккумулятор) в один эффект.
- **`Rage`** (уже компонент → оформить эффектом-контрагентом): длительность; бесстрашие (гасит/блокит Страх) + агрессия (толкает к Атаке, снимает лимит атакующих). Источники: вой вожака/собрата (`Rally`/`EnrageFor`).
- **Холоднокровие = иммунитет к ВНЕШНИМ Страху И Ярости** (эффект не липнет). Врождённая ярость вервольфа — не внешняя, остаётся. Симметрия игрока: холоднокровное сердце → эмоциональная неуязвимость.
- **Рациональность змеи:** змея не «боится» — её психика РАЦИОНАЛЬНО решает «расклад безнадёжен → выхожу» (переход Атака→Настор→отступление по логике, не эффект Страха). Волчий `rout` и змеиный `CheckFlee` СЕЙЧАС похожи, станут принципиально разными: волка ломают эмоцией, змею — только перехитрить.

### Личность особи (расширение `SpawnVariance` на поведение)

Катается раз на спавне, устойчиво (сид, не шум). Малый набор (YAGNI): **храбрость** (порог Страха, уже есть `RollPanicThreshold`), **агрессия** (охотно ли лезет Спок→Настор→Атака), **любопытство** (охотно ли идёт на зацепку). Игрок — детерминирован. «Колесо жизни»: особи в стае ведут себя чуть по-разному.

## Миграция без потери фила (constraint)

Маппинг существующего на новый каркас, тюнинг сохранён:
- **Волк:** `Engaged` → Атака; тревога (`Alerted`)/след-тропление/гремок-любопытство → Настороженность; блуждание → Спокойствие. `rout` → эффект `Fear`. `Rage` остаётся (оформляется эффектом).
- **Змея:** цель-в-термо в упор → Атака; лур/оценка добычи/прокрадывание к тёплому → Настороженность; засада-неподвижность → Спокойствие. `CheckFlee` → рациональное отступление психики (НЕ Страх). Холоднокровна → иммунна к Страху/Ярости уже сейчас.
- Текущие дальности (`sightRange`/`thermalRange`/`scentRange`/`roamSenseRadius`) → база профиля × стартовые множители ≈1, чтобы поведение ≈ нынешнему; рычаг крутим ПОСЛЕ того, как фил подтверждён неизменным.
- **Чекпоинт миграции:** волк и змея по отдельности играются как до S1 (плейтест каждого), затем включаем пер-состоянчатый рычаг и проверяем «прошмыгнуть мимо спокойного».

## Путь расширения (РАДИ ЧЕГО ЗАТЕЯНО — дешёвая добавка нового)

Цель — чтобы новый контент падал ДАННЫМИ, без движка:

- **Новый вид** = ассет `SensoryProfile` (излучение + сенс-строки: дальность/острота/множители по состоянию) + психика маппит 3 состояния на своё поведение (реализует потребитель профиля/`IBodyStatConsumer`). Кода движка — ноль. *Пример-цель: ёж — нюх слабее волчьего по ДАЛЬНОСТИ, но острее по ЧУВСТВИТЕЛЬНОСТИ (берёт тонкий след слабо-пахнущей змеи, который волк пропускает), но след для него быстро выдыхается → так ёж охотится на змею.*
- **Новое чувство** (совиный слух, ежиный тонкий нюх) = ось по рецепту `CONSTRUCTOR_GUIDE` («Добавить новую сенсорику»): сенсор-запрос + эмиттер-сторона → НОВАЯ строка в каждом `SensoryProfile` с безопасным дефолтом (у кого нет чувства — дальность 0; кто не излучает — сигнатура пуста). Существующие виды правок не требуют.
- **Новый эффект** (кровотечение, и будущие) = одна ветка в `HitEffect` (паттерн доказан на Яде/Стане/Страхе).
- **Новый источник Страха/Ярости** (чуждость S4, ловушка, феромон) = просто применяет эффект из словаря; холоднокровие/личность работают автоматически.

Матрица `чувство × состояние × (излучение|восприятие)` — это и есть каркас, в который всё падает ячейками.

## Не-цели S1 (границы)

- Коллективный нюх/триангуляция + эмиссия следа NPC + per-perceiver выдыхание = **S2** (тут только форма профиля под это).
- Кросс-видовой грэб = **S3**. Чужеродность-химеры = **S4**.
- Рендер-сторона восприятия (красивая картинка режимов) = «камера-от-восприятия», запаркована в v1.0 (GDD §5.5). Здесь — сенсор-как-механика.
- Богатое наполнение матрицы значениями (острота, шум) — по мере прихода носителей (ёж/сова), не в S1.

## Чекпоинты (вертикальные срезы)

1. **Каркас:** `AlertState`-компонент + `SensoryProfile`-данные заведены; волк на них (Engaged/Alerted/wander → 3 состояния), играется как до. Плейтест.
2. **Змея на каркасе:** засада/лур/атака → 3 состояния; `CheckFlee` = рациональное отступление; играется как до. Плейтест.
3. **Эффекты:** `Fear` (накопительный, порог храбрости, холоднокровный иммун) заменяет `rout`/`Frighten`/`AddFear`; `Rage` оформлен эффектом. Стая ломается вразнобой; вой игрока пугает; змею не запугать. Плейтест.
4. **Рычаг:** пер-состоянчатые множители включены — «прошмыгнуть мимо спокойного волка», настороженный находит. Личность (агрессия/любопытство/храбрость) разбрасывает особей. Плейтест.

## Открытые вопросы

Нет — ключевые развилки решены в брейншторме (модель состояний = 3+эффекты; Страх накопительный; матрица-способная модель + ленивое наполнение; холоднокровие = иммунитет к внешним Страху+Ярости+рациональность).
