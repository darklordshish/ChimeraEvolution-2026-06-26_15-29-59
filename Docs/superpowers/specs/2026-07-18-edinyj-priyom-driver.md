# Единый приём: доставка = аугумент, виндап/намерение = тонкий драйвер

**Статус:** дизайн-видение пользователя (2026-07-18), spec-first перед кодом. Крупный рефактор — трогает ВСЕ приёмы.

## 0. Принцип (якорь проекта, доведённый до конца)

«Тело = данные, психика = тонкий код, способность = доставка + эффекты» — но сейчас доставка ДУБЛИРУЕТСЯ: у NPC `WindupAbility`-семейство (`BiteAbility/LeapAbility/ChargeAbility/AntlerAbility`), у игрока параллельное `Player*`-семейство (`PlayerBite/Attack/Kick/Howl/Bellow/Antler/Charge/Constrict`) — **8+ пар одного и того же**. Правило трёх давно перевыполнено.

**Цель:** ОДИН орган = один приём, написанный единожды (физика-исполнение в аугументе). Драйвер (ввод игрока ИЛИ психика NPC) — тонкий: «когда жать». «Все ручки/баранки — в шасси и органах-аугументах».

## 1. Что уходит куда

| Слой | Что | Где живёт |
|---|---|---|
| **Доставка (аугумент)** | ИСПОЛНЕНИЕ приёма: удар/эффекты/геометрия (`Hit`+`HitEffect`). Числа — из органа × мощь | компонент на теле, включён флагом органа |
| **Локомоция во время приёма** | движение тела (разбег тарана, ползок) | ШАССИ / общий контроллер, НЕ доставка |
| **Виндап-телеграф** | замах = окно уворота (честность NPC) | ТОНКИЙ код: психика NPC («телеграфирую → жму»); игрок жмёт мгновенно (отзывчивость) |
| **Намерение** | «когда/по кому» | драйвер: `PlayerInputDriver` (кнопка, forward) / психика (решение, target) |

## 2. Три оси ЭКСПРЕССИИ органа (уточнения пользователя)

Приём выражается не только числами — но и ПОЛНОТОЙ формы:

1. **Мощь** (существует): родство 80→100 → множитель ×1…×2 на числа (урон/сила). Капы овершута — `maxDamageReduction`/`minAtkCooldown`.
2. **Родное шасси** (флаг органа `nativeChassis`): полная форма приёма разблокируется ТОЛЬКО на родном шасси; на чужом — ОГРАНИЧЕННАЯ. Формализует нынешний аугумент-кап. Носитель-эталон — ХВОСТ (§3).
3. **Порог фичи** (идея пользователя 2026-07-19, ОТДЕЛЬНЫЙ слайс — семья ГОЛОСА): целый ЭФФЕКТ прячется за порогом экспрессии/родства — открывается только выше него. Пример: СТАН воя доступен лишь на 100% (игрок-полноценный волк), а волку-NPC (экспрессия ~0.45) остаётся баф (+зов кинам) / дебаф (=страх равной силы), без стана. Не «сколько», а «ЧТО ВООБЩЕ открывается» — консистентно через данные (как резали экспрессию, чтобы ослабить зверей). Носитель — вой, ПОСЛЕ хвоста.

**«Зачем» (север пользователя):** эти оси — часть дата-модели ради ОДНОГО — дёшево генерировать химер-монстров ПРАВИЛАМИ, а не ручным управлением всеми. Сделай механику мира → всё разворачивается само.

## 3. Носитель-эталон: ХВОСТ (захват) — РАЗРЕШЁННЫЙ ДИЗАЙН (2026-07-19)

**Данные:** переименовать орган `Змеиный хвост` → `Хвост` (slot остаётся `Хвост`), добавить флаг `nativeChassis = "Змея"`. Ползание — в шасси (`Тело-хвост`, `chassisOnly`), не трогаем.

### Единая машина захвата (одна на игрока и змею)
Схлопывает `PlayerConstrict` (аугумент, кап ст.2) + машину обхвата `SnakePsyche` (ст.3, полное) в ОДНУ доставку. **Ключ, снявший расхождение (идея пользователя):** ст.3 — НЕ отдельная механика, а «то же, что ст.2, но крепче + чок». Поэтому машина ОДНА, а `nativeChassis` перестаёт быть пустым флагом — ему есть что гейтить (кап 2 vs 3).

- **Стадии 1–2–3:** грип крепчает со временем; стадия сильнее защёлкивает ЖЕРТВУ (`Grabbed`) и замедляет ЗАХВАТЧИКА (цена удержания отдельной частью тела).
- **Само-слоу захватчика (смягчён):** ст.1 → ×0.8 (−20%), ст.2 → ×0.6 (−40%), ст.3 → держит ×0.6 ст.2 (слоу не растёт, хват защёлкнут намертво). *(Было жёстче: 0.55/0.3.)*
- **Ст.3 = чистый партер:** жертва сама не вырвется (только зовёт на помощь) + чок-DoT; захватчик добивает.
- **Кап по `nativeChassis`:** `nativeChassis == body.Chassis.speciesName` → доступна ст.3; иначе кап ст.2, без чок-DoT. «Хвост — не тело змеи»: конструкция тела не даёт идеально обхватить крупную добычу. Гейт по ШАССИ, НЕ идентичности (полу-змей на человечьем шасси до ст.3 не дотянется).
- **«Тащить куда угодно»** = свойство захвата (жертва не сопротивляется). КУДА тащить — решает ПСИХИКА: climb/carry-на-стену остаётся в `SnakePsyche`, просто ПОТРЕБЛЯЕТ «несомую жертву». Из машины захвата логика стены не лезет (разделение доставка↔драйвер, как виндап/локомоция).

### Экспрессия сейчас
Ст.3 игроком НЕДОСТИЖИМА (нет змеиного шасси — post-MVP) → сейчас ст.3 гоняет только змея-NPC; у игрока-хвоста кап ст.2. Полная форма — задел на смену шасси (спека идентичности §2). Унификация окупается: правило готово, «полное удушение на змеином шасси» выпадет фичей по умолчанию, когда доедет смена шасси.

### Реализация — самый аккуратный узел
У змеи грип СПЛАВЛЕН с climb/carry-стейт-машиной (`UpdateClimb`, carry-на-насест). Расплести: захват отдаёт «держу жертву, тащу свободно» → психика двигает змею, жертва едет следом. Драйверы (F-тумблер игрока / решение психики) остаются тонкими.

**Слайсинг хвоста:** H1 данные (rename + `nativeChassis`, реген) → H2 единая машина + `PlayerConstrict` на неё (слоу 0.8/0.6/0.6) → H3 змея на общую машину (расплести грип↔carry) → H4 полировка.

## 4. Крутизна — ТАРАН (второй пример)

- Копыта/`Лосиные ноги` несут доставку тарана. Виндап — психика лося (телеграф Charge); игрок жмёт (едет на рывке — локомоция шасси).
- Схлопывает `ChargeAbility` (NPC) + `PlayerCharge` (игрок).

## 5. Развилки к решению (честно — не тривиально)

- **Локомоция во время приёма:** NPC-чардж двигает себя, игроцкий таран едет на общем рывке. Кто владелец? Предложение: доставка НЕ двигает тело — просит «намерение движения» у контроллера/шасси; NPC-психика ведёт разбег, игрок — рывок.
- **Виндап:** данные органа ИЛИ поведение драйвера? Предложение: виндап-телеграф — тонкий код психики (NPC телеграфит перед `Execute`), игрок `Execute` мгновенно. Тогда доставка = чистое исполнение, честно разделяемое.
- **Прицел:** NPC → target (авто), игрок → forward. Доставка принимает и то, и то.

## 6. Слайсинг (после коммита текущего чекпоинта)

- **U1.** Абстракция `Ability`: `Execute(target|forward)` + числа из органа × мощь. Виндап вынесен в вызывающего.
- **U2.** ХВОСТ-эталон: один орган + `nativeChassis`-флаг, схлопнуть `PlayerConstrict`↔`SnakePsyche`-обхват. Проверить чашис-экспрессию.
- **U3.** ТАРАН: схлопнуть `ChargeAbility`↔`PlayerCharge` (развилка локомоции).
- **U4.** Остальные пары (укус/прыжок/пинок/вой/рёв/рога) по образцу.
- **U5.** Полировка: раскладка кнопок = «намерение», орган решает исполнение (уже начато: E=откидывание, Alt=голос).

## 7. YAGNI / границы

Не ломать честность боя: у NPC виндап ОСТАЁТСЯ (окно уворота), у игрока мгновенность ОСТАЁТСЯ. Унифицируем ИСПОЛНЕНИЕ, не отбирая различие драйверов. Смена шасси (полная форма чужому виду) — post-MVP.
