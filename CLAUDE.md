# CHIMERA — правила работы в репо

Рогалик «по флоу» про учёного, химеризующего своё тело. Unity 6 (6000.4.7f1), URP, новый Input System (биндинги В КОДЕ), AI Navigation 2.0. PC, 18+.

## Документы — источники правды

- `Docs/CHIMERA_GDD_v0.16.md` — GDD (дизайн-решения, экономика, мета). Резолвнутое НЕ переоткрывать.
- `Docs/CONSTRUCTOR_GUIDE.md` — как устроен конструктор существ: три закона, словарь, рецепты (вид/орган/способность/эффект/химера-NPC), тюнинг-карта, гочи. **Читать перед любой работой с видами.**
- `Docs/superpowers/specs/` — спеки слайсов. Новая крупная фича = сначала спека (spec-first), потом код.

## Тандем и роли

- Пользователь — геймдизайнер (идеи, развилки, баланс) и руки в Unity Editor. **Новичок в Unity, хочет учиться**: давать пошаговые инструкции для ручных действий в редакторе («создай объект X, повесь Y, выставь Z») + краткое «зачем». Claude — инженерия: C#, архитектура, порядок работ, git.
- Дизайн-решения на развилках НЕ принимать за пользователя — предлагать варианты с трейдоффами и давать выбрать. Честная критика, не поддакивание.
- Claude не может открыть/запустить Unity: верификация = пользователь запускает Play и репортит. **Не коммитить до подтверждения плейтестом.**

## Воркфлоу

- Вертикальные срезы: игра играбельна после каждого шага. Крупное — фазами/слайсами с чекпоинтами.
- **YAGNI / правило трёх:** механику без «носителей» не строить (пример: мульти-сенс ждал второго вида); абстракцию вытаскивать после 2–3 непохожих примеров, не раньше.
- Коммиты: на русском, тематически раздельные (фича ≠ фикс ≠ рефактор), пушить на GitHub после коммита.
- Тюнящиеся системы — объектом в сцену (сериализованные поля видны в инспекторе); авто-бутстрап только для ненастраиваемого.
- После правок `SpeciesBootstrap` — просить пользователя прогнать «Chimera → Создать дефолтные виды». Префабы пересобираются генераторами («Chimera → Создать префаб X»), не руками.

## Unity-гочи (проверено на этом проекте)

- Unity 6 API: `FindAnyObjectByType` (не `FindFirstObjectByType`), `FindObjectsByType<T>()` без параметра сортировки.
- Конструктор `InputAction` — БЕЗ параметра `expectedControlLayout`.
- Перекомпиляция во время Play: несериализованные поля обнуляются → NRE/пропавшие меню. Это НЕ баг кода — стоп Play, пересборка, новый Play. Хардить защитами не нужно (в билдах не бывает).
- Переименование скриптов — ТОЛЬКО `git mv` пары `.cs` + `.cs.meta` (GUID сохраняется, ссылки сцен/префабов живы). Файлы двигать внутри Unity или вместе с `.meta`.
- `UnityEvent`-поля инициализировать `= new()` — иначе null при `AddComponent` в рантайме.
- Цвета NPC запечены в материалы (генераторами); рантайм-тинт (`MaterialPropertyBlock`) дерётся с `Telegraph` — он только у игрока.
- Editor-код (`[MenuItem]`, `SerializedObject`) — в `Scripts/Editor/`.

## Архитектурные якоря (детали — в CONSTRUCTOR_GUIDE)

- **Тело = данные** (`CreatureBody` + `SpeciesSO`: шасси + доноры + экспрессия), **психика = тонкий код** (решения поверх общих «глаголов тела»), **способность = доставка + эффекты** (`WindupAbility`-доставки, `Hit`/`HitEffect`-словарь).
- Статусы — компонентами (`Rage`, `Venom`, `Stagger`, `ColdBlooded`, `Camouflage`), опрашиваются, не шарят состояние.
- Родство начисляет само тело (флаг-скан видов на смерть); `AffinityTracker` — словарь «вид → счётчик».
- Отладка (G/K/N, DebugHud, Dev-панель) — намеренно живёт до настоящего UI, не чистить без запроса.
